# 简历优化器小程序 — 需求文档（完整版）

> 本文档 = 原始需求 + 开发确认的实现方案，作为项目唯一事实来源。

---

## 一、项目概述

| 项目 | 内容 |
|---|---|
| 名称 | 简历优化器 |
| 定位 | 帮你拿更多面试机会的简历工具 |
| 核心目标 | 帮助用户提升简历通过率，并验证用户付费意愿 |
| AI 能力 | DeepSeek（简历优化 + 问题诊断 + 匹配评分）|
| 支付策略 | 冷启动阶段暂不接入支付，采用「预约解锁」模式 |
| 用户标识 | 仅记录 openid（最简方案）|

---

## 二、定价策略（冷启动）

| 方案 | 价格 | 说明 |
|---|---|---|
| 免费体验 | 1 次 | 展示 30% 结果 + 2 条问题 |
| 单次解锁 | 1.9 元 | 解锁本次完整结果 |
| 日卡会员 | 6.9 元 | 24 小时不限次数 |

**后续调价条件**：连续 3 天有收入 或 日收入 > 50 元 → 升级至 3.9 / 9.9 元

---

## 三、用户完整流程

```
输入简历 → AI 优化 → 结果页（锁定状态）→ 制造焦虑 → 点击解锁
                                                                    ↓
                                                          预约弹窗（诱导）
                                                                    ↓
                                                              预约成功
                                                                    ↓
                                                          页面立即解锁
                                                                    ↓
                                                    展示完整对比 + 完整简历
                                                                    ↓
                                                          一键复制使用
                                                                    ↓
                                                              再优化（复用）
```

## 四、页面功能

### 4.1 输入页（首页）

| 元素 | 说明 |
|---|---|
| 主标题 | 让你的简历 **更容易拿到面试**（"更容易拿到面试"蓝色高亮）|
| 副标题 | AI优化 · 问题诊断 · 匹配评分 |
| 目标岗位 | 输入框，placeholder：请输入目标岗位（如：前端开发工程师）|
| 简历内容 | textarea，placeholder：请粘贴你的简历内容（支持从Word/PDF复制）|
| 一键粘贴 | 输入框右上角按钮，读取剪贴板自动填入 |
| 输入引导 | 建议包含：工作经历 / 项目经验 / 技能 |
| 隐私提示 | 内容仅用于优化，不会存储或泄露 |
| 字数统计 | 显示字数，<100 红色警告，>2000 橙色提示 |
| 信任文案 | 已有 X 人完成简历优化（数字蓝色加粗）|
| 主按钮 | 立即优化简历（蓝色渐变，未填完置灰）|
| 辅助文案 | 30秒生成优化结果 |
| 底部标签 | 智能优化 \| 问题诊断 \| 匹配评分 |

**校验逻辑**：
- 岗位为空 → 提示"请输入目标岗位"
- 简历为空 → 提示"请粘贴简历内容"
- 内容 < 100 字 → 弹窗二次确认"建议补充工作经历或项目经验"
- 内容 > 8000 字 → 弹窗二次确认"内容过长，可能影响优化效果"

**使用次数拦截**：
- 第 1 次：正常调用 AI
- 第 2 次起（未预约用户）：点击「立即优化简历」直接弹出预约弹窗

---

### 4.2 结果页

#### 锁定状态（默认）

| 模块 | 展示内容 |
|---|---|
| 评分 | 圆形评分 + 焦虑文案（<70：当前评分偏低，简历较难通过初筛）|
| 优化预览 | **1 条**优化前后对比（before 删除线 / after 截断 / 红色高亮关键词）|
| 锁定提示 | 完整优化内容已锁定 / 还有 X 条未查看（影响面试通过率）/ [解锁完整优化] |
| 优化后简历 | 仅展示 30% + 底部渐变遮罩 + 锁定覆盖层 + [解锁完整优化] |
| 存在问题 | 展示前 2 条 + 红色锁定入口"还有 X 个关键问题未查看（可能直接影响面试结果）"|
| 优化说明 | 完整展示 |

#### 解锁状态（预约后）

| 模块 | 展示内容 |
|---|---|
| 评分 | 大分数展示 + "已达到面试推荐水平"（绿色）|
| 优化总结 | 强化成果表达 / 增加量化数据 / 提升岗位匹配度 |
| 优化前后对比 | **全部 2-3 条**对比（before 删除线 / after 完整 + 关键词红色高亮 / 优化说明）|
| 优化后简历 | **完整展示** |
| 一键复制 | [一键复制简历] 按钮 + "建议粘贴到Word后微调格式再投递" |
| 存在问题 | **展示全部** |
| 优化说明 | 完整展示 |
| 复用引导 | "再优化一版简历" + "建议针对不同岗位分别优化，多数用户会优化2-3版" + [立即再优化] |

**解锁触发**：点击「解锁完整优化」或「查看更多问题」→ 预约弹窗 → 预约成功 → 页面立即解锁

**关键词高亮规则**：AI 返回 `after` 时用 `【】` 包裹关键词（如 `主导【核心模块】开发，提升【30%】系统性能`），前端自动解析为红色加粗。

### 4.3 预约解锁弹窗

| 元素 | 内容 |
|---|---|
| 标题 | 你的简历还有致命问题未修复（红色）|
| 副标题 | 当前简历很可能过不了筛选 / 完整优化后可明显提升通过率 |
| 已完成 | 基础优化（30%）/ 初步问题识别（绿色圆点）|
| 未解锁 | 核心问题未修复 / 面试通过率未提升（红色圆点）|
| 预约说明 | 当前为内测阶段 / 完成预约后优先解锁完整优化功能 / 第一时间开放使用 / 可持续使用优化能力 |
| 价格锚点 | 单次 1.9 元（弱化）/ 全天 6.9 元（推荐，蓝色边框卡片）|
| 预约激励 | 免费解锁1次 或 专属5折优惠 |
| 紧迫感 | 内测优惠即将结束（仅剩少量名额）（红色）|
| 行为引导 | 多数用户会优化2-3版简历，建议直接解锁全天使用 |
| 社会证明 | 已有 X 人预约解锁完整版 |
| 主按钮 | 立即预约解锁（蓝色渐变）|
| 次按钮 | 再看看（弱化灰色）|
| 关闭按钮 | 关闭（右上角文字，弱化）|

## 五、技术架构

### 5.1 前端页面

```
pages/
  index/          # 输入页
  result/         # 结果页（锁定/解锁双状态）
  admin/          # 数据看板
components/
  bookingModal/         # 预约解锁弹窗
  bookingSuccessModal/  # 预约成功弹窗
```

### 5.2 云函数

| 云函数 | 功能 |
|---|---|
| `resumeOptimize` | 调用 DeepSeek API，返回优化结果 + keyComparisons |
| `createReservation` | 创建预约记录（防重复）+ 查询统计 |
| `trackEvent` | 数据埋点（events 集合）|
| `getUserTag` | 用户标签管理（get / update）|
| `getDashboard` | 统计面板（转化率计算）|

### 5.3 数据库集合

| 集合 | 字段 | 用途 |
|---|---|---|
| `reservations` | _openid, source, createTime | 预约记录 |
| `users` | _openid, tag, usageCount, firstVisitTime, lastVisitTime | 用户标签 |
| `events` | _openid, eventType, page, extraData, createTime | 埋点事件 |

### 5.4 AI 返回数据结构

```json
{
  "matchScore": 75,
  "optimizedResume": "优化后的完整简历（纯文本，可直接复制使用）",
  "problems": [
    { "type": "空话套话", "description": "具体问题描述" },
    { "type": "无成果", "description": "具体问题描述" }
  ],
  "optimizationNotes": "优化说明",
  "keyComparisons": [
    {
      "before": "原文典型句子",
      "after": "优化后表达，用【】包裹关键词",
      "explanation": "一句话解释为什么更好"
    }
  ]
}
```

## 六、埋点事件

| eventType | 触发时机 |
|---|---|
| `submit_resume` | 点击「立即优化简历」|
| `show_modal` | 弹窗展示 |
| `click_unlock` | 点击「解锁完整优化」或「查看更多问题」|
| `click_book` | 点击「立即预约解锁」|
| `click_cancel` | 点击「再看看」或关闭弹窗 |

## 七、用户标签流转

```
normal（普通用户）
  → 点击解锁 / 查看更多 / 第二次使用 → clicked_pay（点击过付费）
    → 点击立即预约解锁 → booked（已预约）
```
