# ✅ 简历优化器小程序 - 上线检查清单

> 所有阶段开发完成后，按此清单逐项检查，确认无误后再提交审核

---

## 一、云开发环境配置

- [ ] 已开通微信云开发
- [ ] `miniprogram/app.js` 中 `env` 已填入正确的云开发环境 ID
- [ ] 云开发数据库已初始化（集合会在首次调用时自动创建）

---

## 二、云函数部署清单

需要在微信开发者工具中，对每个云函数右键 → **创建并部署：云端安装依赖**

| 云函数 | 状态 | 说明 |
|---|---|---|
| `quickstartFunctions` | ⬜ | 原有云函数（可选）|
| `resumeOptimize` | ⬜ | 阶段一：DeepSeek AI 简历优化 |
| `createReservation` | ⬜ | 阶段二：预约记录 + 统计查询 |
| `trackEvent` | ⬜ | 阶段三：数据埋点 |
| `getUserTag` | ⬜ | 阶段三：用户标签管理 |
| `getDashboard` | ⬜ | 阶段三：统计面板 |

---

## 三、DeepSeek API Key 配置（可选）

如果不配置，会使用 Mock 数据运行，不影响流程测试。

- [ ] 登录 [DeepSeek 开放平台](https://platform.deepseek.com/) 获取 API Key
- [ ] 云开发控制台 → 云函数 → `resumeOptimize` → 版本与配置 → 环境变量
- [ ] 添加变量：`DEEPSEEK_API_KEY` = `sk-xxxxxxxxxxxxxxxxxxxxxxxx`

## 四、页面功能测试

### 4.1 输入页（首页）

- [ ] 页面标题显示「简历优化器」
- [ ] 岗位输入框正常输入
- [ ] 简历输入框正常输入，字数统计正确
- [ ] 内容超过 5000 字显示警告提示
- [ ] 内容超过 8000 字提交时弹出二次确认
- [ ] 未输入时「开始优化」按钮置灰
- [ ] 输入合法后按钮高亮渐变
- [ ] 点击「开始优化」后显示「AI优化中...」

### 4.2 AI 调用与结果页（第一次使用）

- [ ] 首次点击「开始优化」正常调用云函数
- [ ] 网络异常时提示「网络异常，请检查网络后重试」
- [ ] 成功跳转结果页
- [ ] 结果页显示匹配度评分（圆形卡片）
- [ ] 结果页显示优化后简历（截断展示，约 30%）
- [ ] 结果页底部有「🔓 解锁完整优化」按钮
- [ ] 结果页显示 2 条存在问题
- [ ] 结果页显示「还有 X 条关键问题未展示」入口
- [ ] 结果页显示优化说明

### 4.3 预约弹窗（结果页触发）

- [ ] 点击「解锁完整优化」弹出预约弹窗
- [ ] 弹窗展示：标题 / 已完成 / 未解锁 / 价格锚点 / 预约激励 / 紧迫感 / 社会证明
- [ ] 弹窗显示当前预约人数（动态）
- [ ] 点击「🔥 立即预约」→ 调用云函数 → 显示成功弹窗
- [ ] 点击「再看看」→ 关闭弹窗
- [ ] 点击右上角 ✕ → 关闭弹窗
- [ ] 预约成功后再次点击解锁，不再弹窗（直接提示已预约）

### 4.4 第二次使用拦截（输入页触发）

- [ ] 首次使用后，本地 storage 记录 usageCount = 1
- [ ] 第二次点击「开始优化」时，直接弹出预约弹窗（不调用 AI）
- [ ] 弹窗样式与结果页一致
- [ ] 预约成功后，可正常使用优化功能

### 4.5 清除缓存（测试用）

- [ ] 长按首页标题「帮你拿更多面试机会的简历工具」
- [ ] 弹出确认框「确定要清除本地测试数据吗？」
- [ ] 确认后清除 usageCount / has_booked / optimizeResult
- [ ] 清除后可重新体验首次流程

### 4.6 数据看板

- [ ] 首页底部「📊 数据看板」可点击进入
- [ ] 看板显示：总用户数 / 提交简历数 / 解锁点击数 / 预约人数
- [ ] 看板显示转化率指标（带颜色判断）
- [ ] 看板显示用户分布条形图
- [ ] 点击「刷新数据」可更新

## 五、数据库集合检查

首次运行后，检查云数据库是否已创建以下集合：

| 集合名 | 用途 | 检查项 |
|---|---|---|
| `reservations` | 预约记录 | 已预约用户的 openid、source、createTime |
| `users` | 用户标签 | tag / usageCount / firstVisitTime / lastVisitTime |
| `events` | 埋点事件 | eventType / page / createTime |
